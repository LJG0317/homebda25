About — 요약

데이터를 통해 비즈니스 문제를 정의하고, 의사결정에 직접 연결되는 인사이트를 전달합니다. 분석 과정과 해석을 중시하며, 실무 시나리오 중심의 프로젝트 경험을 보유하고 있습니다.

핵심 역량

  • 데이터 전처리·ETL (Python, SQL)
  • 통계적 분석 및 모델링 (Logistic Regression 등)
  • 대시보드 설계 및 시각화 (Tableau, matplotlib)

업무 방식

  • 문제 정의 → 가설 수립 → 분석 → 시각화 → 리포트
  • 비즈니스 임팩트 중심의 결과 제시

Featured Projects — 문제·접근·결과

각 프로젝트는 문제 정의(Problem), 접근(Approach), 결과(Result), 사용 기술(Stack), 실무 적용 링크(Link)를 명확히 제시합니다.

정렬/강조:

온라인 커머스 매출 분석 및 고객 세분화

Problem: 재구매율 정체로 평균 주문 빈도 감소, 고객 유지 필요.

Approach: SQL로 거래/고객 요약 테이블 생성, Python(pandas)로 RFM 분석, 세그먼트 기반 기여도 분석 및 Tableau 대시보드 제작.

Result: 상위 20% 고객이 매출의 60% 기여. 잠재 우수 고객군 도출 및 카테고리별 이탈 위험 포착.

Stack: Python, SQL, Tableau

Link: Repository (예시)

매장별 매출 성과 분석 및 운영 개선

Problem: 매장 간 매출 편차로 인한 운영 불균형.

Approach: POS/방문 데이터 통합, Excel 피벗으로 초기 인사이트 도출, SQL로 KPI 계산, Tableau로 지역 패턴 시각화.

Result: 객단가가 매출 차이에 주요 영향. 특정 프로모션은 지역 의존성이 높아 지역 맞춤화 필요.

Stack: Excel, SQL, Tableau

Link: Repository (예시)

고객 이탈 분석 및 예측 (기초 모델링)

Problem: 고객 이탈률 증가 우려, 조기 식별 필요.

Approach: 행동 로그 기반 요약 테이블, 특징 비교, 로지스틱 회귀 모델로 기초 예측 수행.

Result: 활동 감소가 주요 예측 변수. 재활성화 캠페인을 통해 초기 리텐션 개선 가능성 확인.

Stack: Python, SQL, Excel

Link: Repository (예시)

상품 추천 실험 분석

Problem: 추천 시스템 도입 후 전환율 효과 측정 필요.

Approach: A/B 테스트 설계, 전환률 비교, 통계적 유의성 검정, 세그먼트별 효과 분석.

Result: 평균 전환률 +3.4% 향상(유의), 특정 카테고리에서 강한 효과 관찰.

Stack: Python, SQL

Link: Repository (예시)
